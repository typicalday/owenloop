# PR3b Build Plan: `idle` Trigger + Alarm Machinery

## Goal

Add the `idle` firing trigger and all time/alarm machinery to oweflow, as specified
in design doc §7.2–7.6 and the E-* decisions in the approved proposal. This is the
time half of the completion evaluator: PR3a landed the clock-free half (`on:`,
`allGreen`, bootstrap exclusion, trigger-cause). This PR adds `idle`, `idleAfter`,
`alarm_at`, `last_progress`, `setAlarm`/`clearAlarm`, the heartbeat re-arm, and the
E-DUE `nextAlarm` query.

Baseline: 351 tests pass on `main` at `93f1cd6`.

---

## Design decisions already resolved (do NOT re-open)

- **E-PURITY**: `src/model.ts` stays pure. `now`, `alarmAt`, `lastProgress`,
  `inFlight` (boolean) are all explicit parameters. No `Date.now()`, no `nowMs()`,
  no timers in `model.ts`. The clock is read only in `engine.ts`.
- **E-GRAMMAR**: `idle` joins `on:`; `idleAfter: "30m"` is a NEW loop-level field
  (parsed via `parseDurationSecs`, same as `cadence`). idle-without-idleAfter = hard
  error; idleAfter-without-idle = hard error.
- **E-ALARM**: Two layers: relative (`last_progress + idleAfterMs`) and absolute
  override (`alarm_at`). Eligibility: `now >= (alarm_at ?? last_progress +
  idleAfterMs)` AND not-done AND quiescent AND not-in-flight.
- **E-PROGRESS**: `last_progress` is workflow-level. Derive it as `MAX(artifact.updated_at)`
  for this workflow. An explicit `last_progress` column is NOT needed — `artifact.updated_at`
  is already reliably stamped by `putArtifact` on every state change (the `updated_at = at`
  in the ON CONFLICT UPDATE). Add `Store.lastProgressMs(workflow): number` querying
  `SELECT MAX(updated_at) FROM artifact WHERE workflow = ?`, with a fallback of 0 if no
  artifacts exist yet.
- **E-OUTCOME**: ONE `outcome` output, re-armable both ways. allGreen re-arm already
  in PR3a. idle re-arm = heartbeat: after an idle firing greens `outcome`, the alarm
  is consumed (cleared). If the evaluator sets a new alarm (setAlarm), that alarm
  elapses → `outcome` re-arms → idle fires again. No alarm = fires once, rests.
- **E-SETALARM**: Engine methods `setAlarm(workflow, loop, at: number)` and
  `clearAlarm(workflow, loop)` writing persisted `alarm_at` on the `task` table.
- **E-DUE**: `Engine.nextAlarm(workflow, now): { dueAt: number | null; isDue: boolean }`.
  Reports the earliest pending time-trigger among idle evaluators in the workflow.

```typescript
getAlarm(workflow: string, loop: string): number | undefined {

  return t?.alarmAt;

/** Persist an absolute alarm time for an idle evaluator loop. */
setAlarm(workflow: string, loop: string, at: number): void {
  // Task row must exist (it is materialized during tick/pendingOwed). If it does not
  // yet exist, upsert a placeholder idle row so alarm_at can be stored.
  const id = taskId(workflow, loop, '');
  const existing = this.getTask(workflow, loop, '');
  if (existing) {
    this.db.prepare('UPDATE task SET alarm_at = ?, updated_at = ? WHERE id = ?')
      .run(at, nowMs(), id);
  } else {
    // Rare: evaluator loop has never been ticked. Insert a minimal idle row.
    this.putTask({ workflow, loop, key: '', status: 'idle', attempts: 0, alarmAt: at });
  }
}

/** Clear the alarm (set alarm_at = NULL). */
clearAlarm(workflow: string, loop: string): void {
  const id = taskId(workflow, loop, '');
  this.db.prepare('UPDATE task SET alarm_at = NULL, updated_at = ? WHERE id = ?')
    .run(nowMs(), id);
}

/**
 * Derive last_progress as MAX(artifact.updated_at) for the workflow.
 * Returns 0 if no artifacts exist yet.
 */
lastProgressMs(workflow: string): number {
  const row = this.db
    .prepare('SELECT MAX(updated_at) AS t FROM artifact WHERE workflow = ?')
    .get(workflow) as { t: number | null };
  return row.t ?? 0;
}
```

**Why task table for alarm_at instead of a new table**: keying alarm per (workflow,
loop) maps directly onto the existing UNIQUE(workflow, loop, key) task row with key='',
which is exactly what idle evaluators use (they are always key=''). This avoids a
new table and reuses the existing upsert path. The only wrinkle is that `setAlarm`
must be willing to INSERT the task row if it doesn't exist yet (see above).

**Why derive last_progress from artifact.updated_at**: every artifact mutation goes
through `putArtifact`, which stamps `updated_at = nowMs()`. This reliably captures
"when did the most recent state change happen" without adding a new column. It
includes runs being born-rejected or rejected — all forward-progress events. This
satisfies E-PROGRESS "derive from an already-recorded timestamp." If the query ever
becomes a bottleneck, the existing `artifact_wf` index on `workflow` covers it.

### Step 4 — model.ts: idleEligibility pure helper + idle branch + idle heartbeat re-arm

**Key constraint**: `eligibleFirings` currently takes `(def: WorkflowDef, arts: ArtifactMap)`.
The idle trigger needs `now`, `lastProgress`, `alarmAt`, and a quiescence/in-flight
boolean. These must be threaded as explicit parameters.

**Approach**: add an optional `TimeFacts` parameter bag to `eligibleFirings` so the
signature change is additive and back-compat (a caller not passing it gets no idle
firings, which is correct — no idle loops without these facts):

```typescript
export interface TimeFacts {
  /** Current time (ms since epoch). */
  now: number;
  /** Timestamp of the most recent artifact state change for this workflow. */
  lastProgressMs: number;
  /** Whether any run is genuinely in flight (claimed + lease fresh). */
  inFlight: boolean;
  /** Per-loop alarm_at values: loop name → alarm_at in ms (if set). */
  alarms: Map<string, number>;
}

export function eligibleFirings(
  def: WorkflowDef,
  arts: ArtifactMap,
  time?: TimeFacts,
): Firing[]
```

Add a pure helper `idleEligible`:
```typescript
/**
 * Returns true if the idle trigger is eligible for this loop.
 * Pure — takes only explicit parameters; no Date, no nowMs.
 *
 * Eligibility: now >= (alarm_at ?? last_progress + idleAfterMs)
 *   AND workflow is NOT done (has debts excluding evaluator outputs)
 *   AND workflow is quiescent (no active inputsGreen/allGreen firings — separate concern,
 *       verified by the engine before calling eligibleFirings with inFlight=false)
 *   AND no run is in-flight for this loop
 *
 * "Not done" is the NOT-allGreen gate: idle fires when there are non-evaluator debts.
 * When the workflow IS all-green, allGreen owns it — idle must not fire (that
 * would cause idle AND allGreen to race; allGreen is the winner for clean done).
 */
function idleEligible(
  loop: LoopDef,
  arts: ArtifactMap,
  time: TimeFacts,
): boolean {
  const { now, lastProgressMs, inFlight, alarms } = time;
  if (inFlight) return false; // R12: in-flight ≠ idle

  const idleAfterMs = loop.idleAfterMs!; // validated: idle requires idleAfter
  const alarmAt = alarms.get(loop.name);
  const threshold = alarmAt ?? (lastProgressMs + idleAfterMs);
  if (now < threshold) return false;

  // Compute evaluator's own outputs to exclude from all-green check.
  const evaluatorOutputs = new Set<string>(plainOutputs(loop));

  // Not-done gate: idle fires only when there ARE non-evaluator debts.
  // (If the workflow is all-green → allGreen fires, not idle.)
  const wfAllGreen = allArtifactsGreen(arts, evaluatorOutputs);
  if (wfAllGreen) return false; // allGreen owns this condition

  return true;
}
```

**Idle heartbeat re-arm in `maintainDecisions`** — mirror the allGreen-rearm block:

The idle re-arm fires when: the evaluator's `outcome` IS green AND the idle alarm has
elapsed again (a new alarm was set by setAlarm after the last fire). But `maintainDecisions`
is pure and only sees `arts`, not `time` or `alarms`. So the re-arm must also accept
`TimeFacts` optionally, or the re-arm must be handled at the engine level.

**Decision**: add an optional `time?: TimeFacts` to `maintainDecisions` signature.
If provided, check: for each idle evaluator with a green outcome, if
`idleEligible(loop, arts, time)` returns true (i.e., a new alarm has elapsed), emit
a `reject` re-arm op on `outcome`. This is the heartbeat: the alarm elapses →
`outcome` re-arms → `eligibleFirings` offers the idle firing again.

```typescript
export function maintainDecisions(
  def: WorkflowDef,
  arts: ArtifactMap,
  time?: TimeFacts,
): CascadeOp[]
```

Add after the existing allGreen-rearm block:
```typescript
// idle heartbeat re-arm: if an idle evaluator's outcome is green AND the idle
// threshold has elapsed again (alarm set by the evaluator), re-arm outcome.
if (time) {
  for (const loop of def.loops) {
    const triggers = resolvedTriggers(loop);
    if (!triggers.includes('idle')) continue;
    if (!idleEligible(loop, arts, time)) continue; // threshold not reached

    const evaluatorOutputPaths = plainOutputs(loop);
    for (const path of evaluatorOutputPaths) {
      const art = arts.get(path);
      if (!art || art.acceptance !== 'green' || art.terminal) continue;
      ops.push({
        kind: 'reject',
        path,
        reason: 'idle-rearm: alarm elapsed again — re-evaluating idle condition',
      });
    }
  }
}
```

**No-thrash guarantee**: `idleEligible` returns false when `now < threshold`.
After the idle alarm fires and the engine clears it (`clearAlarm`), `alarms` no
longer contains this loop, so `threshold = lastProgressMs + idleAfterMs`. Unless
a new alarm is set or `lastProgressMs + idleAfterMs` elapses, `idleEligible` is
false → no re-arm. Stable.

**Quiescence note**: `inFlight` in `TimeFacts` is computed by the ENGINE (not model.ts)
from the claimed-task scan (same logic as `claim`'s freshness check). The engine
computes this once before calling `eligibleFirings`, passing it through. The model is
pure and never reads the clock.

### Step 5 — engine.ts: thread time facts, alarm clear-on-fire, setAlarm/clearAlarm/nextAlarm, TickResult.dueAt

**`tick` — thread `TimeFacts` into eligibleFirings**:

```typescript
tick(workflow: string, opts: { now?: number } = {}): TickResult {
  const def = this.defFor(workflow);
  const now = opts.now ?? nowMs();
  return this.store.tx(() => {
    this.settle(workflow, def, now);   // settle also needs now for idle re-arm
    const reaped = this.reap(workflow, now);

    const arts = this.artMap(workflow);

    // Compute time facts for idle eligibility (clock-read boundary).
    const timeFacts = this.computeTimeFacts(def, workflow, arts, now);

    const firings = eligibleFirings(def, arts, timeFacts);
    const { selected, deferred } = this.applySchedule(workflow, def, firings, now);

    // Clear alarm_at for any idle firing that was selected (consume the alarm).
    for (const f of selected) {
      if (f.cause === 'idle') {
        this.store.clearAlarm(workflow, f.loop);
      }
    }

    const orders: Order[] = [];
    const allDeferred: DeferredFiring[] = [...deferred];
    for (const f of selected) {
      const result = this.claim(workflow, def, f, arts, now);
      ...
    }

    // E-DUE: compute earliest pending time-trigger for the result.
    const dueAt = this.computeDueAt(def, workflow, arts, now);

    return { workflow, orders, reaped, deferred: allDeferred, dueAt };
  });
}
```

**`computeTimeFacts` private method**:
```typescript
private computeTimeFacts(
  def: WorkflowDef,
  workflow: string,
  arts: ArtifactMap,
  now: number,
): TimeFacts {
  const lastProgressMs = this.store.lastProgressMs(workflow);
  const inFlight = this.isInFlight(workflow, now);
  const alarms = new Map<string, number>();
  for (const loop of def.loops) {
    if (!loop.on?.includes('idle')) continue;
    const alarmAt = this.store.getAlarm(workflow, loop.name);
    if (alarmAt !== undefined) alarms.set(loop.name, alarmAt);
  }
  return { now, lastProgressMs, inFlight, alarms };
}
```

**`isInFlight` private method** — mirrors the `claim` freshness check:
```typescript
private isInFlight(workflow: string, now: number): boolean {
  for (const task of this.store.listTasks(workflow)) {
    if (task.status !== 'claimed') continue;
    const run = task.run ? this.store.getRun(task.run) : undefined;
    const fresh =
      !!run &&
      run.outcome === undefined &&
      (task.claimedAt === undefined || now - task.claimedAt <= this.reapTtlMs);
    if (fresh) return true;
  }
  return false;
}
```

**`settle` — pass `now` and thread into `maintainDecisions`**:

`settle` is called many times but currently takes `(workflow, def)`. It needs `now`
for the idle re-arm check in `maintainDecisions`. Approach: thread `now` as an
optional third parameter; when not provided, fall back (all internal non-tick callers
can pass it through from their outer `opts.now ?? nowMs()` read, or default to 0
which means idle re-arm never fires during a non-time-aware settle — acceptable
because idle only re-arms in the tick path).

```typescript
private settle(workflow: string, def: WorkflowDef, now?: number): void {
  const limit = 1000;
  for (let i = 0; i < limit; i++) {
    let arts = this.artMap(workflow);
    const owed = pendingOwed(def, arts);
    for (const a of owed) this.store.putArtifact({ ...a, workflow });
    if (owed.length) arts = this.artMap(workflow);

    // Only pass TimeFacts when we have a clock reading (tick path).
    // Non-tick settles (green, reject, etc.) never trigger idle re-arm.
    let timeFacts: TimeFacts | undefined;
    if (now !== undefined) {
      timeFacts = this.computeTimeFacts(def, workflow, arts, now);
    }

    const ops = maintainDecisions(def, arts, timeFacts);
    for (const op of ops) this.applyOp(workflow, def, arts, op);

    if (owed.length === 0 && ops.length === 0) return;
  }
  throw new Error(`settle did not converge for ${workflow} (possible cascade cycle)`);
}
```

All existing callers of `settle(workflow, def)` remain valid (now is optional).
Only `tick` passes `now`.

**`nextAlarm` / `computeDueAt` for E-DUE**:
```typescript
/**
 * Returns the earliest pending time-trigger among idle evaluators for this workflow,
 * and whether it is due at `now`. Used by an external scheduler to decide when to
 * wake this instance.
 */
nextAlarm(workflow: string, opts: { now?: number } = {}): { dueAt: number | null; isDue: boolean } {
  const now = opts.now ?? nowMs();
  const def = this.defFor(workflow);
  const arts = this.artMap(workflow);
  const lastProgressMs = this.store.lastProgressMs(workflow);
  let earliest: number | null = null;

  for (const loop of def.loops) {
    if (!loop.on?.includes('idle')) continue;
    // Skip if outcome is already rested (green) and no alarm pending — no wake needed.
    const alarmAt = this.store.getAlarm(workflow, loop.name);
    const threshold = alarmAt ?? (lastProgressMs + (loop.idleAfterMs ?? 0));
    if (earliest === null || threshold < earliest) earliest = threshold;
  }

  return {
    dueAt: earliest,
    isDue: earliest !== null && now >= earliest,
  };
}
```

**Alarm clear-on-fire**: done in `tick` (above) before calling `claim`. When an idle
firing is selected, `clearAlarm(workflow, loop.name)` is called. This prevents the
alarm from re-triggering on the next tick without the evaluator explicitly setting a
new one. The heartbeat re-arm then only happens when a NEW alarm elapses.

### Step 6 — docs/design.md: extend §21

- **§21.8 `idle` trigger** — fires when quiescent past threshold (not done, not in-flight,
  now >= threshold). Threshold = alarm_at if set, else last_progress + idleAfter.
  Carries cause:'idle'.
- **§21.9 Sliding window (relative alarm)** — `idleAfter: "30m"` in YAML. Every

- **§21.10 Absolute alarm (override)** — `alarm_at` stored on the task row for
  (workflow, loop, key=''). Survives process restart. Takes precedence over the
  relative fallback. Set via `engine.setAlarm(wf, loop, at)`, cleared on fire.

- **§21.12 Heartbeat re-arm** — when idle fires and greens `outcome`, if the evaluator
  calls setAlarm in its body, the next tick re-arms `outcome` (idle-rearm) when the
  alarm elapses. Without a new alarm, idle fires once then rests (no thrash).
- **§21.13 Purity discipline** — model.ts receives `now`, `lastProgressMs`, `inFlight`,
  `alarms` as explicit parameters in `TimeFacts`. No `Date`, no `nowMs()` in model.ts.
  The clock is read only at the engine boundary (`opts.now ?? nowMs()`). `eligibleFirings`
  and `maintainDecisions` are idempotent for a fixed (arts, now) pair.

## Edge Cases to Call Out

**Restart persistence of alarm_at**: `alarm_at` lives in the `task` table (SQLite),
not in memory. A process crash and restart reads it back via `getAlarm` in
`computeTimeFacts`. The migration ensures existing databases get the column.

**No-thrash**: `idleEligible` returns false when `now < threshold`. After idle fires
and the alarm is cleared, `threshold = lastProgressMs + idleAfterMs`. Unless
`lastProgressMs` is old enough that `now >= lastProgressMs + idleAfterMs` is already
true, the idle trigger rests. If it is already true (long-running quiescent workflow),
the evaluator must call `setAlarm(wf, loop, now + delta)` in its body to push the
next fire out — the design explicitly supports this.

**In-flight ≠ idle (R12)**: `isInFlight` checks all claimed tasks, not just the
evaluator. Any fresh claimed run (any loop) in the workflow blocks idle. This is correct
per the proposal: "don't fire idle while a lease is genuinely in flight, just slow."

**Idle vs allGreen mutual gating**: `idleEligible` returns false when `allArtifactsGreen(arts,
evaluatorOutputs)` is true (the workflow is done). allGreen and idle are mutually exclusive
owners of their respective conditions. A loop declaring `on: [allGreen, idle]` could in
theory get both in the same tick — but since idle gates on NOT all-green and allGreen gates
on all-green, they are structurally disjoint. This is implicit in the current code paths and
should be asserted in tests.

**Quiescence and allGreen/inputsGreen firings**: The `inFlight` field covers in-flight
leases (claimed tasks). However, the proposal also says "no eligible inputsGreen/allGreen
firing" as a quiescence condition. In practice: if there IS an eligible inputsGreen firing,
it will be claimed in the same tick, making `inFlight = true` by next tick. The in-flight
check is the simpler, correct guard. Adding an explicit "no eligible non-idle firings" check
would add complexity with no practical benefit: the tick claims them immediately.

**allGreen-rearm and idle-rearm coexistence**: a loop declaring `on: [allGreen, idle]` has
BOTH re-arm paths active. They are structurally disjoint (allGreen-rearm triggers when NOT
all-green; idle-rearm triggers when all-green is false AND threshold elapsed). No conflict.

**`settle` convergence with idle**: `maintainDecisions` may emit an idle-rearm op for a
loop whose `outcome` is green and the alarm has elapsed. After applying the op (outcome →
owed), the next iteration of settle's loop: `idleEligible` is still true but `outcome` is
now owed (a debt) → `maintainDecisions` will NOT emit another idle-rearm (the art is not
green). Convergence holds.

**idle evaluator with no artifacts in workflow yet**: `lastProgressMs` returns 0 if no
artifacts. `threshold = 0 + idleAfterMs`. If `now >= idleAfterMs`, idle fires. This is
correct: a workflow that never made progress is quiescent from the start. In practice,
`pendingOwed` materializes artifacts on the first settle, so `updated_at` exists.

**Task row may not exist yet when setAlarm is called by a worker**: The store-level
`setAlarm` handles this with an UPSERT (creates the task row if absent). The engine-level
`setAlarm` is a pure wrapper around the store method.
